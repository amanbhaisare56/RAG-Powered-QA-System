# Smart Q&A API

A production-like Node.js/Express backend that answers user questions using the **RAG (Retrieval-Augmented Generation)** pattern — retrieving relevant documents from MongoDB and generating grounded, structured responses via an LLM.

---

## 🚀 Features

- RAG-based question answering using LLM (Groq + LangChain)
- Context-aware responses grounded in MongoDB documents only
- JWT authentication & protected routes
- Per-user rate limiting (10 req/min)
- Structured logging with Winston (userId, latency, confidence)
- Zod schema validation for LLM output
- Bonus: `/api/ask/history` endpoint + Docker support

## How It Works

```
POST /api/ask
  → JWT middleware        (401 if invalid)
  → Rate limiter          (429 if > 10 req/min)
  → ragService            extract keywords → score MongoDB docs → top 3
  → deriveConfidence()    score ≥ 6 → high | ≥ 3 → medium | else → low
  → LangChain prompt      inject docs as context, restrict LLM to context only
  → Groq LLM              generate answer
  → Zod validation        enforce { answer, sources[], confidence }
  → Save to QAHistory     + Winston log (userId, latencyMs, confidence)
  → 200 OK
```

> Confidence is derived from keyword retrieval scores, not generated by the LLM — making it deterministic and hallucination-proof.

## API Reference

### Auth

| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/auth/register` | Register a new user |
| POST | `/api/auth/login` | Login — returns JWT |

### Documents

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/docs` | List all knowledge base documents |

### Q&A (JWT required)

| Method | Endpoint | Rate Limit | Description |
|---|---|---|---|
| POST | `/api/ask` | 10 req/min | Ask a question — returns RAG answer |
| GET | `/api/ask/history` | — | Last 10 Q&A pairs for the user |

#### Request

```json
POST /api/ask
{ "question": "What is the refund policy?" }
```

#### Response

```json
{
  "answer": "Refunds are processed within 5-7 business days...",
  "sources": ["69d559a70c34837c52b15189"],
  "confidence": "high"
}
```

## Sample cURLs

```bash
# Register
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password123"}'

# Login (copy token from response)
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password123"}'

# Ask a question
curl -X POST http://localhost:5000/api/ask \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"question":"What is the refund policy?"}'

# Get history
curl http://localhost:5000/api/ask/history \
  -H "Authorization: Bearer YOUR_TOKEN"
```
